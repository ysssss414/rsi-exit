# RSI卖点信号识别器 v0.1 技术说明

## 1. 范围与成功标准

本版只处理 A 股日线趋势股的卖点风险识别。它不做首次选股、主线识别、分时、自动下单、参数寻优或收益回测。验收优先级是：计算口径正确、高点合理、分类可解释、日期因果正确、输出可人工复核。

## 2. 数据与前复权

统一入口：

```python
AmazingDataAdapter.get_daily_bars(
    symbol,
    start_date,
    end_date,
    adjust="forward",
    force_refresh=False,
)
```

已复用接口：

- `AmazingDataProvider.login`
- `BaseData.get_calendar`
- `BaseData.get_code_info`
- `MarketData.query_kline`
- `BaseData.get_backward_factor`

原始日 K 先按请求区间缓存。前复权价格按查询终点归一化：

```text
forward_price[t] = raw_price[t] * backward_factor[t] / backward_factor[end]
```

只调整 OHLC；成交量和成交额保留 AmazingData 原始值。因子缺少起始覆盖时直接报错，不用未来因子反向填补。日期升序、重复日保留最后一行、OHLC/成交字段缺失或非法时停止；不插入停牌日。

AmazingData 1.1.6 的 operator 模块只有 pyc，真实 Numba 对 `cache=True` 无法定位源码。适配层可启用与旧项目隔离运行时相同的 no-JIT 装饰器兼容模式；它只影响 SDK 导入，不改变本项目数值算法。SDK 日 K 转换需要 Pydantic v2，复权因子 HDF5 缓存需要 PyTables。

SDK 会话在全部输出写完后释放。异常不会被改写成空数据或伪成功。

## 3. RSI14

实现：

```text
LC = REF(CLOSE, 1)
RSI = CN_SMA(MAX(CLOSE-LC, 0), 14, 1)
    / CN_SMA(ABS(CLOSE-LC), 14, 1) * 100
Y[t] = (X[t] + 13 * Y[t-1]) / 14
```

- `seed_mode=first`：第一个非空 X 直接作为递推种子，是默认口径；
- `seed_mode=mean`：累计 14 个非空 X 后用算术均值初始化，再进入同一递推；
- X 缺失时当行输出缺失，但保留上一个递推状态；流水线本身拒绝缺失 OHLC，因此该行为主要用于函数级诊断；
- 分母为 0 时 RSI 保持缺失，不把无波动强行解释成 0 或 100。

注意：`first` 的早期 RSI 对查询起点敏感。当前验收严格以 2026-02-01 请求区间为种子起点；与行情软件全历史序列人工对比时，应额外获取更长预热区间并只截取展示区间。这是下一版优先改进，不应通过改公式掩盖。

## 4. 无未来高点状态逻辑

每个 t 只做三步：

1. t 满足收盘价和 RSI 同步上升，且都是 t-2..t 的最大值，成为候选；
2. 只有 t+1 收盘价和 RSI 都低于 t，才在 t+1 收盘后确认；
3. `peak_date=t`、`confirm_date=signal_date=t+1`、`earliest_action_date=t+2`。

检测器不读取 t+2 或更晚的数据来判断 t。独立回撤只使用前一代表高点之后、当前候选之前已经发生的数据。

同波段合并使用显式活动高点状态：不足最小交易日间隔，或中间既没有 RSI 回落 5 点也没有价格回撤 2.5%，只生成合并审计记录。价格明显更高时更新代表高点；价格在容差内时只在 RSI 更高时更新。已生成的历史信号行不回写，后续比较从确认后的新代表高点继续。

`peaks.csv` 保留每个已确认候选：独立高点为 `is_independent_peak=true`；合并候选填写 `merged_into_peak_id`，若它更新了代表高点则 `canonical_updated=true`。

## 5. 背离、重置与动能锚

四类信号严格按相邻有效高点划分：趋势强化、顶部背离、降低高点弱反弹、价格降低但 RSI 改善。弱反弹不增加背离次数。

背离周期在以下条件重置：趋势强化、RSI 高点明显提高、间隔超过 30 个交易日、中间 RSI 低于 50、状态进入 S3。重置后的首个有效高点建立新基准，不伪装成一背。

动能锚是当前周期 RSI 最高的有效代表高点；v0.1 只输出日期、价格、RSI 和相对值，不额外触发仓位动作。

## 6. 状态机保守解释

- 初始状态设为 S0；RSI 未形成时保持状态并输出 `RSI_UNAVAILABLE`；
- S3/S4/S5 阶段不假定已经自动重新买入，因此 S4 仓位上限继续为 0；
- S5 只恢复买入资格，仓位上限允许回到 1，但不表示自动买入；
- S4 或 S5 修复失败时回到 S3；
- 第三次连续顶部背离强制 S3；
- 每日最终上限取基础状态与持续背离风险上限的较小值。较宽松的新风险信号不会自动抬升上限，只有明确周期重置才解除旧背离上限。

## 7. 已保留的规则歧义

### RSI 容差下边界

原始需求同时写了：

- RSI 明显降低：`R_i <= R_prev - tolerance`；
- 顶部背离：`R_i < R_prev - tolerance`。

两者在恰好低 1 点时冲突。v0.1 优先采用更具体的信号分类公式和更保守的卖出解释：恰好低 1 点仍记为容差内持平，不触发背离。单元测试固定了该边界。若以后决定改成闭区间，只需统一修改 `classify_peak_pair` 和对应测试，不改其他流程。

### 新周期明确开始

需求没有提供独立的 `new_cycle` 外部字段。v0.1 只使用可验证的已有重置条件；S3 进入后清空周期，S5 本身不重复清空。

### S5 后续持有语义

S5 表示“恢复资格”而非实际持仓。只要 RSI>=70 且价格在 MA20 上方就保持 S5；60..70 且仍在 MA20 上方回到 S4，而不是假定已经持仓后进入 S1。

## 8. 中际旭创验收解释

在线数据：2026-02-02 至 2026-07-20，共 110 个交易日，前复权。

默认参数结果：

- 14 个独立波段高点；
- 2 次顶部背离：2026-05-29、2026-06-05 确认；
- 3 次弱反弹；
- 2026-07-20 状态 S3，RSI 约 38.8449，目标仓位上限 0。

1416.88 是 2026-06-22 的盘中最高价；规则使用收盘价，因此程序记录同日收盘高点 1382.33 为 P0015，并在 6月23日双下降确认，最早 6月24日执行。它没有被计为第三次连续背离：6月8日状态已经进入 S3，按明确重置条件清空了背离周期，P0015 成为新周期首个动能基准。确认日状态是 S4_REPAIR_WATCH，仓位上限仍为 0，因此最高价区域已处在严格风险约束下。没有为迎合 1416.88 写任何代码特例。

## 9. 当前已知问题与下一步

1. RSI 与行情软件精确对比前应增加可配置预热历史，并在输出中区分计算区间与展示区间。
2. AmazingData 1.1.6 登录日志会由原生 SDK 输出较多连接元数据；本项目日志不输出账号或密码，但无法完全控制原生日志格式。
3. 在线异常发生在 SDK 原生进程退出之前时，Python 无法保证执行 finally；应继续让真实下载保持独立短进程。
4. 批量入口和汇总契约已具备，第二批股票尚未在线批量验收。
5. v0.1 只验证信号语义，不证明收益；下一步应先完成人工盲核和行情软件 RSI 对值，再做组合回测或参数敏感性分析。
